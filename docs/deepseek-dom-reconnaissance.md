# DeepSeek DOM 人工侦察指南（Milestone 2B-0.1）

本工具只用于收集和扩展适配器所需的结构证据。Milestone 2B-1 已有最小普通文字问答适配，但推理、停止、重生成、编辑、文件和联网引用仍必须先通过本流程取得证据。不要在真实历史对话中操作；请新建只含合成、无个人信息、无上传文件的对话。工具不读取 Cookie、Web Storage、网络请求、账号资料或完整页面正文。

## 准备与授权

1. 构建并加载解压缩扩展，打开 `https://chat.deepseek.com/`，由用户自行登录。
2. 新建对话，只输入预先编写的合成问题；不要打开历史对话或上传文件。
3. 打开扩展 popup，展开“DeepSeek DOM 侦察（开发工具）”。默认状态下工具不运行。
4. 勾选“我已启用侦察开发模式”，点击“主动授权 DeepSeek 页面”。浏览器会请求可选的 `https://chat.deepseek.com/*` 权限。
5. 点击“导出结构快照”。输出只包含标签、class、`data-*`、`aria-*`、层级、文本长度、子节点数量和脱敏后的 URL 路径结构。

未授权、当前页面不匹配或注入失败时，工具安全停止，不采集任何内容。侦察完成后可在浏览器扩展设置中撤销该可选权限。

## 单节点导出与脱敏

1. 点击“选择单个对话节点”，阅读并确认隐私警告。
2. 返回页面并移动鼠标。红色半透明 overlay 会显示当前候选的标签、ID、最多两个 class、尺寸和父级深度；overlay 使用 `pointer-events: none`，不会修改目标节点样式。
3. 从回答中的文字开始悬停，使用滚轮向上或 `ArrowUp` 逐级扩大到父节点；滚轮向下或 `ArrowDown` 返回之前的子节点。滚动页面或缩放窗口后，高亮会同步位置。
4. 当高亮完整覆盖一条助手消息或用户消息时，按 `Enter` 或鼠标左键确认；按 `Escape` 取消。`html/head/body/script/style` 不可直接选择。
5. 重新打开 popup。它会显示已选节点的标签、class、文本长度、HTML 字符数、子节点数量、选中时间和脱敏来源域名，并提供不含正文的可复制调试摘要。
6. 如果看到“当前可能只选中了句子或最内层文本元素”，点击“重新选择”并向上扩大；如果看到“当前范围可能过大”，重新选择并向下缩小。
7. 确认范围合适后点击“读取待脱敏 HTML”，再次确认隐私警告。需要放弃时点击“清除选择”，页面内存中的原始 `outerHTML` 会立即删除。
8. 原始 HTML 只显示在 popup，并仍只保存在当前页面内存；扩展不下载、不写文件、也不把原始 HTML 写入 `chrome.storage`。
9. 手工复制到**仓库外**的临时文件。禁止把原始 HTML 放进 `apps/extension/fixtures/deepseek`。
10. 由人工从结构快照确认用户节点和助手节点选择器，再运行：

```powershell
node .\apps\extension\tools\sanitize-deepseek-fixture.mjs `
  --input C:\private-temp\deepseek-node.html `
  --output .\apps\extension\fixtures\deepseek\ordinary-completed.html `
  --user-selector "<已确认的用户节点选择器>" `
  --assistant-selector "<已确认的助手节点选择器>"
```

脚本删除 `script/style/svg/img` 等活动或媒体节点、事件属性，清除链接查询参数，替换账号/文件名/对话 ID 属性和消息正文，并生成仅含计数的 `.report.json` 结构差异报告。发现邮箱、手机号、URL token、残留文件名或长随机字符串时返回非零退出码且不生成 fixture。输出文件已存在时也拒绝覆盖。

最后必须人工检查 HTML 和报告：正文是否全为固定合成文本、是否仍有昵称/标题/文件名/账号标识、结构是否足以支撑候选选择器。确认后才可更新选择器报告。

### 选择快捷键

| 操作 | 效果 |
| --- | --- |
| 鼠标移动 | 切换最内层候选并立即刷新 overlay |
| 滚轮向上 / `ArrowUp` | 扩大到可选择的父节点 |
| 滚轮向下 / `ArrowDown` | 返回刚才的子节点 |
| 鼠标左键 / `Enter` | 确认当前高亮节点并保存结构化摘要和 `outerHTML` |
| `Escape` | 取消并彻底移除 overlay 与事件监听器 |

调试摘要只包含 DOM 路径、标签结构、属性名、直接子元素标签和文本长度，不包含属性值或正文。选中记录包含的 `data-*`、`aria-*` 值和原始 `outerHTML` 仍只存在于 DeepSeek 页面对应的扩展隔离内存中。

## 统一记录模板

每种状态都填写以下字段；没有证据时写“未确认”，不得猜测。

| 字段 | 要记录的证据 |
| --- | --- |
| 用户节点特征 | 标签、候选 `data-*`/`aria-*`、相对层级和文本长度变化 |
| 助手节点特征 | 标签、候选属性、与用户节点/轮次容器的关系 |
| 最终答案节点 | 是否独立节点、代码和表格的保留方式 |
| 推理节点 | 是否与最终答案分离、折叠后是否仍在 DOM |
| 生成中状态 | 明确状态属性/按钮/标志及其出现和消失时机 |
| 完成状态 | 可观测的明确完成标志；没有则记录文本稳定行为 |
| DOM 变化方式 | 新增、原位修改、整体替换或节点重排 |
| 稳定属性 | 跨刷新、重生成和多个合成对话仍一致的语义属性 |
| 易变属性 | 哈希 class、序号、文本、会话 ID、`nth-child` 等 |

## 十一种场景

每个场景至少保存生成前、生成中、完成后三份结构快照；涉及状态切换的场景还要记录 Mutation 类型和节点是否保持同一引用。

### 1. 普通问答

使用一问一答的合成学习问题。定位轮次边界、用户节点、助手节点和最终答案节点；验证候选属性在刷新后仍存在。记录完成标志，以及答案是原位更新还是一次性插入。

### 2. 流式回答

选择会持续输出的合成问题，在开始、输出中和完成时分别取快照。记录文本是否修改同一文本节点、分段追加子节点或替换整个答案；记录生成中标志和完成标志的精确变化。流式内容不得导出为正式 fixture 前的“已完成”状态。

### 3. 带推理区的回答

确认推理区和最终答案是否为独立节点、相对顺序、折叠/展开的影响，以及最终答案是否可能在推理结束后才出现。记录适配器应忽略的推理节点边界，不记录推理正文。

### 4. 代码块回答

使用仅含无敏感信息的短代码示例。记录 `pre/code` 等语义标签是否保留、语法高亮是否产生易变 class、复制按钮是否位于答案正文内，以及复制按钮变化是否会误判新回答。

### 5. 表格回答

使用合成 Markdown 表格。记录最终 DOM 是 `table` 结构还是其他表示、表头与单元格层级，以及渲染后无关控件变化。不要以表格正文作为选择器。

### 6. 多轮对话

连续提交至少三轮合成问答。记录轮次容器边界、用户与助手如何配对、新轮次是追加还是导致旧轮次重建，以及是否存在稳定轮次属性。不得把真实会话 ID 写入报告。

### 7. 停止生成

流式期间点击停止。记录生成中标志如何消失、停止后的答案节点是否标为不完整、是否出现可区分的停止状态，以及后续 DOM 是否仍修改。安全默认是证据不足时不采集。

### 8. 重新生成回答

对同一合成问题触发重新生成。记录旧助手节点被替换、隐藏、保留为候选版本还是原位清空；记录 generation 边界及无稳定标志时的安全失败方式。

### 9. 编辑问题后重新回答

编辑合成问题再提交。记录原用户节点和助手节点是否被替换、轮次标识是否变化、旧版本是否仍留在 DOM。新问题与新答案必须被视为新事件，不依赖位置序号。

### 10. 切换历史对话

仅在本次侦察创建的两个合成对话间切换。不要展开或记录其他历史标题。记录主对话容器是替换还是原位重绘、旧 MutationObserver 目标是否失效，以及公开 URL 路径仅应记录形状而非会话 ID。

### 11. 刷新页面

刷新当前合成对话。记录相同轮次的 DOM 是否重建、候选语义属性是否保持一致，以及如何避免刷新后重复发送。事件稳定性最终应来自规范化问答哈希，而不是 DOM ID。

## 隐私检查清单

- 只使用全新合成对话，不包含个人姓名、邮箱、手机号、学校、账号、真实课程内容或文件。
- 结构快照中没有消息正文、昵称、历史标题、上传文件名或完整会话路径。
- 原始单节点 HTML 位于仓库外，脱敏完成后立即按用户自己的安全流程清理。
- 脱敏脚本零敏感发现，且人工复核 HTML 与差异报告均通过。
- Git 变更中只出现脱敏 fixture 和结构计数报告；不出现截图、数据库或原始导出物。
- 候选选择器必须在多个合成场景、刷新和重生成中复核，不能只依赖单个自动生成 class。
